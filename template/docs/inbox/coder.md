# Inbox — coder(写码方收件箱 · 抢占区 + FIFO 双区)

> 只放待办。条目 = 指针 + 摘要,全文见对应 docs/ 产出目录;历史在 git log。
>
> **取任务顺序(收到「读收件箱,开工」时依次判断)**:
> 1. **阻塞门**:`gh pr list --state open` 自查——存在自己未合入的实现 PR(客座 docs PR 除外)→ 只能继续该 PR 的整改/等待复审,不得取任何区的新任务。口令不解除阻塞,合并才解除。
> 2. **抢占区非空** → 取抢占区**首条**(即使 FIFO 有更早的任务)。抢占区写入权:审阅方(合并后发现的整改/回归/热修);设计方仅限注明理由的插队(如全队受益的基础设施)。
> 3. 以上皆空 → 取 FIFO **队首**,不得跳取。
> 4. 取任务/读回执即删条目,随首个 commit 提交;设计方永远只追加 FIFO 队尾;审阅方的在审 PR 整改**不进本收件箱**(整改跟着 open PR 走,阻塞门已保证其独占)。
