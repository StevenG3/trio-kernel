# Inbox — coder(写码方收件箱 · FIFO 任务队列)

> 只放待办。条目 = 指针 + 摘要,全文见对应 docs/ 产出目录;历史在 git log。
>
> **FIFO 队列规则**:
> 1. 本收件箱是先进先出队列:任务自上而下排列,**只能取队首任务**,不得跳取。
> 2. **阻塞门(最高优先规则)**:响应「读收件箱,开工」前,先跑 `gh pr list --state open` 自查——存在自己未合入的实现 PR 时队列阻塞:不得取任何新任务,只能继续该 PR 的整改/等待复审。口令不解除阻塞,合并才解除。
> 3. 取队首任务时删除该条目,随开工首个 commit 提交;审阅回执类条目读完即删。
> 4. 设计方可随时向**队尾**追加任务,与开发进行中的 PR 并行,互不影响。
